SSMS (Stationary Shop Management system)

Desktop Application Development

A desktop-based Stationary Shop Management System built with Python and SQLite. This project included user authentication, product and stock management, sales and purchase processing, customer and supplier management, and automated report generation.

Client: Fahad Stationary Shop
Date: April 2025
Technologies: Python, Tkinter, SQLite3, MySQL, Figma
Project URL: www.example.com

Case Study

The client needed a desktop-based solution to manage their stationary shop more efficiently, as manual record-keeping was causing 30% errors in stock tracking and delays in report generation.

I designed and developed a full-featured Stationary Shop Management System using Python (Tkinter) and SQLite. The system included user authentication with roles, product and stock management, sales and purchase processing, customer and supplier tracking, and automated report generation in PDF format.

The result was a 60% improvement in inventory accuracy, a 50% reduction in manual workload, and a 40% faster report generation process compared to the client's previous manual methods.

Key Milestones

Requirements analysis and project planning
Database design and ERD modeling
UI/UX design with Tkinter prototypes
Backend development with Python and SQLite
Implementation of role-based authentication
Development of product, stock, sales, and purchase modules
Integration of automated PDF report generation
Testing and bug fixing
Final deployment and documentation
